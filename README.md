## ? 项目概述

本项目是一个针对资源受限的 Cortex-M4 微控制器设计的高可靠、安全固件在线升级（IAP）系统。

传统单片机 IAP 升级极易因弱网环境、异常断电或电磁干扰导致设备“变砖”，且明文传输存在固件被窃取和篡改的风险。本系统采用**异地更新（Out-of-place）架构**，引入外部 SPI Flash（W25Q128）作为缓冲与备份区，结合纯软件 AES-128 CTR 流式加解密，提供了一套具备**防变砖双区备份、自动/手动回滚、流式解密及严格状态机校验**的工业级固件升级解决方案。

## ?? 软件架构 (Software Architecture)

本系统采用严格的分层解耦架构设计，自下而上分为硬件层、驱动层、中间件层、核心业务层与应用层。低层对高层透明，保证了代码的高可移植性与模块化。

```text
| +--------------------+  +--------------------+  +-------------------+ |
| |  FW Manager        |  |  Protocol Stack    |  |  Security & Check | |

| |  回滚/元数据维护)  |  |  控制/指令路由)    |  |  校验/Magic Header) | |
| +--------------------+  +--------------------+  +-------------------+ |
+-----------------------------------------------------------------------+
|                         中间件层 (Middleware Layer)                   |
| +-----------------+ +------------------+ +-------------+ +----------+ |
| |  AES-128 CTR    | |  CRC16 / CRC32   | | RingBuffer  | |EasyLogger| |

```

## ? 核心特性

- **?? 极致的容灾与防变砖设计（双区备份 & 回滚）**
  - 基于外部 W25Q128 Flash 实现 A/B 双区固件备份。
  - **自动回滚：** 启动时严格检测应用区完整性，若监测到 APP 连续 3 次启动失败，底层主动触发防变砖机制，自动从备份区恢复固件。
  - **手动回滚：** 支持通过上位机发送专用指令（`0x86`）进行无缝手动版本回退。
- **? 端到端固件安全防护**
  - **加密传输：** 采用 AES-128 CTR 模式，固件以密文形式分片传输及存储。
  - **流式解密与烧录：** 支持边读、边解密、边烧录内部 Flash，在极低内存开销下防止固件逆向。
- **? 高健壮性自定义通信协议栈**
  - 底层基于 `RingBuffer` 与串口中断实现“生产-消费”异步解耦。
  - 构建严格的 5 状态字节流状态机（帧头 → 操作码 → 长度 → 载荷 → CRC16），加入 20ms 帧间超时重置机制，彻底免疫非理想链路下的数据帧断裂与字节丢失。
- **? 内存安全的 Flash 极客操作**
  - 针对 Flash 的擦写特性，采用 4KB 静态缓冲区处理块读写，通过内存复用彻底杜绝栈溢出（Stack Overflow）。
  - 所有 Flash 写入操作（内外 Flash 及元数据）均强制执行 **Read-back Verify（写后回读比对）**，保障物理烧录 100% 可靠。
- **? 零依赖纯净跳转**
  - 使用汇编指令直接操作 MSP 与 PC 寄存器，在严格的资源清理（DeInit）与 VTOR 重定向后，实现瞬时安全跳转。

## ? 内存分区规划 (Memory Map)

本系统将存储划分为内部运行区与外部备份区，解耦了下载过程与烧写过程：

### 1. 内部 Flash 规划 (STM32F407 - 512KB)
| 扇区 (Sector) | 地址范围 | 大小 | 功能描述 |
| :--- | :--- | :--- | :--- |
| Sector 0~2 | `0x0800 0000 - 0x0800 BFFF` | 48 KB | **Bootloader 运行区** |
| Sector 3 | `0x0800 C000 - 0x0800 FFFF` | 16 KB | **APP 参数区** (含魔数、固件大小、CRC32) |
| Sector 4~7 | `0x0801 0000 - 0x0807 FFFF` | 448 KB | **APP 应用程序运行区** |

### 2. 外部 SPI Flash 规划 (W25Q128 - 16MB)
| 区域 | 大小 | 功能描述 |
| :--- | :--- | :--- |
| **Metadata Zone** | 4 KB | 存储分区激活状态、回滚标志、启动失败计数等元数据 |
| **Zone A** | 512 KB | 固件备份区 A（密文存储，最大支持 512KB 空间分配） |
| **Zone B** | 512 KB | 固件备份区 B（密文存储，最大支持 512KB 空间分配） |

## ? 通信协议规范

基于 UART 的类 Modbus-RTU 异步通信帧结构。

**通信参数：** `115200 bps, 8 Data bits, No Parity, 1 Stop bit`

**数据帧格式：**

| 字段 | 帧头 (Header) | 操作码 (Opcode) | 长度 (Length) | 载荷 (Payload) | 校验 (CRC16) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **字节数** | 1 Byte | 1 Byte | 2 Bytes (Little Endian) | `N` Bytes | 2 Bytes (Little Endian) |
| **说明** | 固定 `0xAA` (下发)<br>`0x55` (响应) | 指令类型 | `Payload` 的长度 | 数据体 (Max: 4096+8) | 包含 Header 到 Payload 的 CRC16 校验 |

**核心 Opcode 定义：**
- `0x84 (FW_WRITE)`: 传输加密固件分片（带总体 Size 与 Chunk Offset）。
- `0x85 (FW_COMMIT)`: 提交固件，触发全量 CRC32 验证及内部 Flash 解密烧写。
- `0x86 (FW_ROLLBACK)`: 手动触发固件回滚。
- `0x22 (BOOT)`: 校验完毕，拉起 APP 程序。

## ?? 构建与开发指南

### 依赖环境
- **IDE**: Keil MDK-ARM v5 / VSCode + GCC
- **MCU**: STM32F407VET6 (或兼容的 Cortex-M4 芯片)
- **第三方组件**: `EasyLogger` (日志系统), `RingBuffer`

### 编译与下载
1. 克隆本仓库：`git clone https://github.com/yourusername/iap-bootloader.git`
2. 使用 Keil 打开 `mdk/stm32f407.uvprojx`。
3. 编译工程，确保无 Error 及 Warning。
4. 使用 ST-Link/J-Link 将生成的二进制文件烧录至 `0x08000000`。

### 触发 Bootloader 升级模式
系统支持以下三种策略进入升级模式：
1. **引导期拦截：** 每次上电有 3 秒引导等待期，期间串口接收到任何合法数据帧即进入升级模式。
2. **异常拦截：** APP 校验失败或发生崩溃（魔数丢失/CRC错误），系统强制停留在此模式。
3. **硬件按键触发：** 按住指定按键（Key3）长达 3 秒触发强制刷机。
